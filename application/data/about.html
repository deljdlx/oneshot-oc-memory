Objectifs

Au delà de l'aspect démonstration technique, ce jeu a pour objectif de pouvoir potentiellement servir de support de formation.
Dans la version courante voici les points pouvant être abordés grace à ce jeu :

  • Architecture
    • Architecture "microservice"
    • Services assimilés REST
  • Backend
    • Autoloading
    • Structure PSR 4
    • Protocole HTTP et Routing
    • Héritage de classe
    • SQL (avec SQLite3)
  • Front
    • Javascript natif orienté objet
    • Ajax
    • Utilisation des variables CSS
    • Notion d'injection de dépendance
  • Divers
    • Algorithme simple de mélange (permettrait de pouvoir faire une introduction à la complexité algorithmique)
    • Usage de fonctions mathématiques simples pour des question de mise en page (et oui les math peuvent être sympas !)

Remarques

Je suis l'auteur des bibliothèques Phi utilisées coté PHP (bibliothèques/framework encore en cours de conception)

J'ai utilisé une partie de mes bibliothèque pour gagner du temps, notemment sur l'aspect routing et bdd

Pour visualiser le jeu avec une jauge de temps classique modifier le fichier public/application/OC/Memory/javascript/Configuration.js

Améliorations

Liste des points à améliorer

  • Meilleure architecture/intégration CSS
  • Utilisation d'un logiciel de build coté front, par exemple Webpack
  • Utilisation de composer coté PHP
  • Création de taches de maintenances coté backend en utilisant Phing (exemple concernant la tache "reset" qui ne devrait pas être accessible depuis un endpoint
  • Création d'une couche configuration/DI coté php
  • Architecture webcomponent ou React coté Javascript
  • Utilisation d'ES6

Pour aller plus loin, déploiement

Je n'ai pas eu le temps d'aborder le sujet

Mais afin de pouvoir livrer un support de formation "prêt" à l'usage, j'aurai livré une image Vagrant (pas eu le temps de la créer)

Pour aller plus loins dans la logique de déploiement, utilisation d'Ansible

Pour certaine tâches de déploiement utilisation de PHPDeployer

Concernant Docker, j'ai utilisé cette technologie pour faire "poper" des bdd MySQL de démo. Cependant cela fait pas mal de temps que je n'ai pas utilisé cette technologie

Packager l'application dans un fichier phar serait également quelque chose d'intéressant

Considération personnelles

Je suis loin d'être complétement satisfait par la qualité de mon code de la finition de mon livrable. J'ai cependant voulu aborder un maximum de points potentiellement utiles dans le cadre d'une formation de développeur.

Dans le cadre d'un vrai support de cours mon code aurait été beaucoup plus commenté. (avec des rappel vers des leçons)

Je n'ai pas utilisé scss par manque de temps et "d'utilité immédiate". J'avais cependant voulu en montrer l'usage et j'avais commencé à coder quelque chose dans ./application/bin/scss.php
Dans le cadre d'une formation ce script m'aurait servi de support pour montrer l'usage de PHP en ligne de commande

Le choix de développer une jauge de temps en cercle avait pour objectif de montrer quelque chose de joli (pour le petit effet "wow" auprès d'apprenants) mais surtout pour illustrer un usage "flagrant" de l'injection de dépendance. (De plus, montrer à des étudiants qu'avec un peu de math il est possible de faire des choses sympa est plaisant je trouve)
